# ПЕТРГУ ФТИ 6 КУРС | РАЗРАБОТКА RAG
Репозиторий содержит пример реализации RAG системы (Retrieval-Augmented Generation), которая позволяет LLM отвечать на вопросы, используя внешние данные из векторной БД.

## Ход проекта
### Описание
Проект реализован при помощи языка Python.
Используемые библиотеки:
- ChromaDB
- Transformers (HuggingFace)
- python-telegram-bot

***

#### Компоненты кода
- Инициализация БД: Используется ChromaDB (PersistentClient) для хранения фактов в векторном виде.

- Модель: `tiiuae/Falcon-H1-1B-Instruct` — компактная языковая модель для генерации текста.

- Логика RAG (`/rag`):

- - Берет текст пользователя.

- - Ищет 2 наиболее похожих факта в БД.

- - Формирует промпт: `Контекст: {факты} + Вопрос: {текст} = Ответ`.

- Прямой запрос (`/chat`): Обращение к модели напрямую без контекста.

***

#### Описание модели
- Тип модели: только декодер
- Архитектура: гибридная, трансформеры + архитектура Mamba
- Языки: английский, многоязычный (+ русский).
- Выбран для демонстрации работы на CPU.


#### Данные в векторной БД
- Луна - единственный естественный спутник Земли.
- Она находится примерно в 384 400 км от Земли.
- Диаметр Луны - около 3 474 км (примерно в 4 раза меньше Земли).
- Фазы Луны (новолуние, полнолуние и др.) возникают из-за изменения освещенности Луны Солнцем.
- Луна вызывает приливы и отливы в океанах Земли благодаря своей гравитации.
- Луна вращается вокруг своей оси и вокруг Земли за одно и то же время (~27,3 суток). Это называется синхронное вращение, поэтому мы всегда видим одну и ту же сторону.

***

#### Сценарий с использованием БД (Команда /rag)
Запрос: Расскажи что-нибудь о Луне

Процесс: Система извлекает из БД факты о спутнике и диаметре.
Ответ:

Контекст: Луна — единственный естественный спутник Земли. Диаметр Луны — около 3 474 км (примерно в 4 раза меньше Земли)

Ответ: Луна — это одно из двух естественных спутников Земли. Диаметр Луны — около 3 474 км.

Примечание: Здесь мы видим небольшую ошибку модели (про два спутника), но точные цифры взяты из БД.

#### Выводы
1) Модель интегрирует данные, если они поданы в промпте.

2) Без контекста модель Falcon-1B может выдавать случайные токены или заготовленные шаблоны, не относящиеся к теме.

3)  При малом количестве max_new_tokens или слабой модели возможен обрыв текста.
